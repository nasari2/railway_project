Railway Management System (Django Project)

The Railway Management System is a full-featured web application developed using Django and Python, aimed at streamlining railway operations and workforce management. It provides a robust and user-friendly platform for admin and worker roles, enabling efficient handling of tasks, leaves, and employee management through an intuitive dashboard.

Key Features

1. Worker Management
Register new workers with comprehensive details including username, email, phone number, address, and job role.
Maintain a secure and organized database of all workers.
Role-based access ensures only authorized users can perform specific actions.

2. Task Management
Admins can assign tasks to workers with a clear title and status tracking.
Workers can view their assigned tasks, with the ability to update their status.
Tasks are categorized as Pending or Completed, providing clear workflow visibility.

3. Leave Management
Workers can apply for leave with reasons and date ranges.
Admins can approve or reject leave requests directly from the dashboard.
Pending leaves are highlighted, ensuring timely decision-making.

4. Admin Dashboard
Real-time statistics displaying total workers, total tasks, completed tasks, pending tasks, total leaves, and pending leaves.
Interactive dashboard cards allow admins to click and view detailed lists.
Provides a comprehensive overview of workforce and operational status at a glance.

5. Secure Authentication
Role-based login system with Admin and Worker accounts.
Ensures that sensitive operations are restricted to authorized personnel.

6. Responsive User Interface
Designed with Bootstrap and custom CSS for a clean, responsive, and user-friendly experience.
Compatible across devices including desktops, tablets, and mobile phones.

Technology Stack
Backend: Django, Python
Frontend: HTML5, CSS3, Bootstrap
Database: SQLite (configurable to MySQL or PostgreSQL)
Version Control: Git & GitHub
Benefits & Use Cases
Ideal for demonstrating CRUD operations in Django.
Helps organizations manage tasks and workforce efficiently.
Provides hands-on experience with dashboard analytics, role-based access, and leave management workflows.
Suitable for academic projects, prototypes, or small-scale railway operation management.




















































# Railway Management System (Django Project)

The **Railway Management System** is a fully functional web application developed using **Django** and **Python**, designed to streamline railway operations, workforce management, and task tracking. This system allows **admins and workers** to efficiently manage tasks, leaves, and employee data through a user-friendly interface.  

---

## 🌟 Features

### Worker Management
- Register new workers with **username, email, phone number, address, and job role**.
- Role-based access ensures secure and organized data management.
- View all workers and their details in the admin dashboard.

### Task Management
- Assign tasks to workers with clear **title and status**.
- Workers can track and update their task status (**Pending / Completed**).
- Admins can view **total, pending, and completed tasks** in real-time.

### Leave Management
- Workers can **apply for leave** specifying dates and reasons.
- Admins can **approve or reject leave requests**.
- Dashboard highlights **pending leaves** for quick action.

### Admin Dashboard
- Real-time statistics including **total workers, total tasks, completed tasks, pending tasks, total leaves, pending leaves**.
- Clickable cards to view **detailed worker, task, and leave lists**.
- Clean, responsive interface built with **Bootstrap**.

### Secure Authentication
- Role-based login for **Admin and Worker accounts**.
- Only authorized users can perform sensitive operations.

### Responsive Design
- Fully responsive layout for **desktop, tablet, and mobile devices**.
- Custom CSS and Bootstrap for a clean, professional look.
